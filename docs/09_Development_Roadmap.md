## 🛠 Coding Standards & Git Flow

To maintain code quality during collaboration:
- **Coding Standards:** Backend follows **PHP PSR-12** standards, implementing Repository and Service patterns. Frontend uses **TypeScript**, ESLint, and component-driven layouts.
- **Git Flow:** Feature branches use the following format:
  `feature/authentication`, `feature/listings`, `feature/jobs`.
- **Commit Messages:** Follow semantic commit patterns (e.g. `feat: add auth module`, `fix: search bug`).
