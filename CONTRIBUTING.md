# Contributing to HOYAMU.NET

Thank you for your interest in contributing to HOYAMU.NET! To maintain code quality and enterprise standards, please follow these guidelines.

## Development Workflow

1. **Branching**: Branch off the `develop` branch.
   - Feature branches: `feature/your-feature-name`
   - Bugfix branches: `bugfix/your-bug-name`
2. **Coding Standards**:
   - Backend (Laravel): PSR-12 standard compliance.
   - Frontend (Next.js): TypeScript, ESLint, Prettier compliance.
3. **Commit Messages**: Enforce semantic commit prefixes:
   - `feat: ...` for new features
   - `fix: ...` for bug fixes
   - `docs: ...` for documentation changes
   - `refactor: ...` for code restructurings
   - `test: ...` for unit tests modifications
